# Security Policy

## Reporting a vulnerability

Do not disclose security issues in public issues or pull requests.

Report suspected vulnerabilities directly to the organization owners and include:

- A clear description of the issue
- Affected repository or component
- Reproduction steps or proof of concept
- Impact assessment if known

## Response target

Initial triage should happen within 3 business days.

## Handling guidance

- Restrict discussion to the minimum set of maintainers required
- Prepare a fix and validation plan before public disclosure
- Publish remediation notes after the fix is deployed or merged
