# The Resonance Team

The Resonance Team is a product engineering organization focused on shipping reliable web, API, and mobile experiences with a strong emphasis on developer experience, review discipline, and operational clarity.

## What lives here

- Shared engineering standards and repository defaults
- Team handbook and onboarding material
- Platform and infrastructure foundations
- Shared contracts and SDKs for cross-platform delivery

## Working agreements

- Open a pull request for all non-trivial changes
- Keep changes small, reviewable, and linked to a tracked task
- Prefer squash merges into `main`
- Treat CI as a release gate signal even where GitHub Free cannot enforce branch protection on private repositories

## Core repositories

- `team-handbook`: onboarding, processes, runbooks, and repo map
- `platform-infra`: infrastructure, automation, and platform tooling
- `shared-sdk`: shared contracts, SDKs, and cross-platform types
- `.github`: org-wide templates and community health files

## Team structure

Planned teams in this organization:

- `engineering`
- `frontend`
- `backend`
- `mobile`
- `platform`
- `maintainers`

## Contact

For access requests, onboarding, or team operations, use GitHub issues in `team-handbook` or contact the org owners directly.
