# Internationalization

Vietnamese and English as the default language pair for all Resonance Team
products. Vietnam is our primary market; English is the fallback and the
developer language.

## Strategy

Every product ships with **VN + EN** from day one. Adding a third language
later should be a string-import exercise, not a refactor.

## Web (Next.js)

**Library:** `next-intl`

```typescript
// apps/web/src/i18n/request.ts
import { getRequestConfig } from 'next-intl/server';

export default getRequestConfig(async ({ requestLocale }) => {
  const locale = await requestLocale;
  return {
    locale,
    messages: (await import(`../messages/${locale}.json`)).default,
  };
});
```

Messages live in `apps/web/src/messages/{vn,en}.json`.

**Pattern:**
- Route-based locale: `/vn/dashboard`, `/en/dashboard`.
- `<Link href="/dashboard" locale="vn">` for navigation.
- Server Component translations: `useTranslations()`.
- Client Component translations: `useTranslations()` from `next-intl`.

## Mobile (Flutter)

**Library:** `flutter_localizations` + Dart `intl` package.

```dart
// pubspec.yaml
dependencies:
  flutter_localizations:
    sdk: flutter
  intl: any

// lib/l10n/app_vn.arb  ← Vietnamese messages
// lib/l10n/app_en.arb  ← English messages
```

MaterialApp uses `localizationsDelegates` + `supportedLocales: [Locale('vi'), Locale('en')]`.

## Zalo Mini App

ZMP SDK provides built-in i18n via `zmp.i18n`. Messages are key-value
JSON per language in the app config. Share keys with the web app where
possible.

## Backend error messages

API error responses include both languages when user-facing:

```json
{
  "error": "VALIDATION_ERROR",
  "message": {
    "vi": "Email không hợp lệ",
    "en": "Invalid email address"
  }
}
```

The client picks based on the user's locale header or stored preference.
The backend does not decide the language — it responds in both and the
client chooses.

## Date and number formatting

| Surface | Approach |
|---|---|
| Web | `Intl.DateTimeFormat` / `Intl.NumberFormat` (built-in) |
| NestJS | `Intl` (Node 24 has full ICU) |
| Flutter | `intl` package |
| Zalo | ZMP API formatters |

## Content direction

VN + EN are both LTR. If an RTL language is added later, tailwind-rtl
and Flutter's `Directionality` handle it.

## Checklist per new app

- [ ] `messages/vn.json` and `messages/en.json` created.
- [ ] No hardcoded strings in UI components — everything goes through the
  translation function.
- [ ] API error responses include `message.vi` and `message.en`.
- [ ] Date/number formatting uses locale-aware APIs, not `.toString()`.
